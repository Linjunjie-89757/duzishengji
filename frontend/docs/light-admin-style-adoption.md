# 轻量后台视觉规范采纳说明

本项目参考根目录 `风格样式优化完整版.txt`，但按当前 demo 偏好的“轻后台工作台”方向做收敛，避免页面边界过重或卡片套卡片。

## 采纳规则

- 保留 Element Plus 原生体系，不引入新的 UI 框架。
- 保留浅灰页面底、白色工作面、深色侧栏、主蓝 `#409eff`。
- 全局默认弱边框、无阴影或极轻阴影。
- 表格默认更紧凑：表头 `44px`，行高 `48px`。
- Tab 使用蓝色下划线激活，不做胶囊、渐变或强阴影。
- 按钮、输入框、下拉框只做轻 hover 和聚焦反馈。
- 空状态使用 `#fafafa` 底色、居中图标和提示文案。
- 加载态使用 Element Plus 图标式 loading，不使用大面积骨架屏作为默认加载态。

## 收敛规则

- 不强制所有区域包白色卡片。列表、树、筛选、编辑器可以有轻容器；页面大区域优先使用留白和细分割线。
- 不强制所有页面套固定高度。接口自动化这类工作台页面可以固定视口高度，普通列表页保持自然滚动。
- 不强制替换现有 Lucide 图标。新增内容优先 Element Plus 图标，现有图标只统一尺寸、颜色和线宽。
- 不使用 `transition: all`。仅对颜色、边框、阴影、透明度等非布局属性做过渡。
- 不使用厚重阴影、装饰性图标卡、渐变背景或大面积彩色块。

## 页面类型策略

- 工作台/接口编辑器：白色连续工作面 + 细分割线，减少外层圆角和阴影。
- 普通列表页：筛选区、表格区用轻容器，分页贴近表格底部。
- 统计区：保留左色条 + 数字信息块，不增加大图标和强阴影。
- 空页面：大空场景可有图标，小空白块只保留文字。
